HTTP 402 was reserved in RFC 2616 (1999) but never implemented — no standard payment flow existed. Now AI agents need to pay per API call.

Invoica built x402: a protocol extension that returns 402 → agent reads Payment-Required header → completes USDC transfer via EIP-712 signature → retries request.

Real atomic settlement, not invoicing theater.

How it works:
1. Agent hits API without auth
2. Server returns 402 + payment address
3. Agent signs EIP-712 TransferWithAuthorization (USDC on Base)
4. Server detects settlement on-chain
5. Request completes

No human invoice. No NET-30. No reconciliation hell.

This is what 'Payment Required' was supposed to mean in 1999.

Why agents need this: OpenAI API charges $0.002/1K tokens. An agent making 10K calls/day = $20. Manual invoicing breaks at scale.

x402 makes the payment *part of the HTTP flow* — agents budget in code, APIs verify settlement cryptographically.

Invoica handles compliance (VAT reverse charge via base58 memo field) + settlement detection.
